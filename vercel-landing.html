How it works

Push code. That's it.

Three steps from local to live. No YAML pipelines to maintain, no servers to provision.

01

Connect your repository

Link GitHub, GitLab, or Bitbucket. We auto-detect your framework and configure the build.

02

Configure (or don't)

Most projects work with zero config. For edge cases, add a pipeline.json.

03

Deploy and iterate

Every push triggers a build. Preview URLs for branches. Instant rollback if something breaks.

// pipeline.json
{
"framework": "nextjs",
"regions": ["iad1", "hnd1", "fra1"],
"build": {
"command": "npm run build",
"output": ".next"
}
}

Frequently asked questions

Which frameworks are supported?

Next.js, Nuxt, SvelteKit, Astro, Remix, Gatsby, Hugo, Jekyll, and any static site builder. Custom frameworks work via Dockerfile in your repo.

Is there a free tier for open-source projects?

Yes. Verified open-source projects get the Pro plan free. Apply through our Open Source page and we'll get back to you within 48 hours.

Can I use my own domain?

Yes — add it in one click from the dashboard. SSL is automatic via Let's Encrypt. Supports apex, subdomains, and wildcard records.

What counts as an edge invocation?

Every HTTP request to your edge functions. Static asset requests don't count. We reset the counter on the first of each month.
